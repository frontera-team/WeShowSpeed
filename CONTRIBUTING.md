# Алгоритм решения задачи

1. Создание ветки
	1. Имя ветки:
		1. 'feature/'+название вашей задачи. *Пример:* feature/link-to-Max
2. Переход на эту ветку с помощью `git checkout`
3. Создание коммитов в этой ветке
4. Синхронизация вашей ветки с main
	```
	git checkout main
	git pull origin main
	git checkout your-feature-branch
	git rebase main
	```
5. Создание Pull Request на мерж с main
6. Ревью PR и его закрытие

# Правило именование коммита
`<тип>(<область>): <описание>`

## Типы

| Тип          | Когда использовать                                    |
| ------------ | ----------------------------------------------------- |
| **feat**     | Новая функциональность для пользователя               |
| **fix**      | Исправление бага                                      |
| **docs**     | Изменения в документации                              |
| **style**    | Форматирование, отступы, запятые (не меняющие логику) |
| **refactor** | Рефакторинг без изменения поведения                   |
| **test**     | Добавление или исправление тестов                     |
| **chore**    | Обновление зависимостей, настройка CI, тех. работы    |
## Область
Компонент или модуль, который был затронут

Backend:

| Область        | Когда использовать                                           |
| -------------- | ------------------------------------------------------------ |
| **api**        | Общие изменения в API (эндпоинты, маршруты)                  |
| **auth**       | Аутентификация и авторизация (логин, регистрация, JWT, роли) |
| **users**      | Всё, что связано с пользователями (профиль, настройки)       |
| **orders**     | Заказы (создание, оплата, история)                           |
| **payments**   | Платёжные системы, интеграции с PayPal/Stripe                |
| **products**   | Товары, каталог, категории                                   |
| **cart**       | Корзина покупок                                              |
| **database**   | Миграции, схемы, запросы (не связанные с конкретной фичей)   |
| **cache**      | Кэширование (Redis, Memcached)                               |
| **queue**      | Очереди задач (RabbitMQ, Sidekiq, Celery)                    |
| **email**      | Email-уведомления, рассылки                                  |
| **validation** | Валидация входных данных                                     |
| **middleware** | Промежуточные слои (логирование, CORS, rate limiting)        |
| **logging**    | Логирование и мониторинг                                     |
| **config**     | Конфигурация приложения, переменные окружения                |
| **docs**       | API-документация (Swagger, OpenAPI)                          |
| **tests**      | Тесты (если затрагивают конкретную область)                  |
Frontend:

| Область        | Когда использовать                                    |
| -------------- | ----------------------------------------------------- |
| **ui**         | Общие компоненты интерфейса (кнопки, инпуты, модалки) |
| **pages**      | Целые страницы или роутинг                            |
| **components** | Переиспользуемые компоненты (конкретные)              |
| **store**      | Управление состоянием (Redux, Vuex, Pinia)            |
| **api-client** | Клиент для общения с бэкендом (axios, fetch)          |
| **styles**     | Глобальные стили, темы, CSS-переменные                |
| **forms**      | Формы и валидация на клиенте                          |
| **routing**    | Навигация, роутер                                     |
| **hooks**      | Кастомные хуки (React) / composables (Vue)            |
| **utils**      | Утилиты, хелперы                                      |
| **assets**     | Изображения, шрифты, статика                          |
Общее:

| Область      | Когда использовать                             |
| ------------ | ---------------------------------------------- |
| **deps**     | Обновление зависимостей                        |
| **docker**   | Dockerfile, docker-compose                     |
| **ci**       | Настройка CI/CD (GitHub Actions, GitLab CI)    |
| **scripts**  | Скрипты для сборки, деплоя, миграций           |
| **security** | Безопасность (CSP, зависимости с уязвимостями) |
| **perf**     | Оптимизация производительности                 |
| **i18n**     | Интернационализация (переводы)                 |
| **a11y**     | Доступность (accessibility)                    |
| **seo**      | Поисковая оптимизация                          |

Тестирование

| Область         | Когда использовать                   |
| --------------- | ------------------------------------ |
| **unit**        | Модульные тесты                      |
| **e2e**         | Сквозные тесты (Cypress, Playwright) |
| **integration** | Интеграционные тесты                 |

Используйте **слэш для подуровней**: `api/users`, `ui/buttons`, `store/cart`
